CONNECTION MODE
WIRED — USB Serial
Direct USB via Web Serial API. Requires Chrome or Edge. Use for bench testing alongside PlatformIO.
WIRELESS — WebSocket
ESP32 hosts a WebSocket server over WiFi AP. Dashboard connects as WS client. No cable needed in-flight.

⬡ WIRELESS SETUP GUIDE

1.In platformio.ini, add: lib_deps = Links2004/WebSockets @ ^2.4.0
2.In firmware: #include <WiFi.h> and #include <WebSocketsServer.h>
3.In setup(): WiFi.softAP("RocketAP","launch123"); webSocket.begin(); webSocket.onEvent(wsEvent);
4.In loop event handler, call webSocket.loop(); each iteration.
5.Replace Serial.println(line) with webSocket.broadcastTXT(line); — keep the same >KEY:val format, no other changes needed.
6.Connect laptop to WiFi RocketAP, set URL to ws://192.168.4.1:81, flip the toggle, hit Connect.
7.For longer range: use ESP32 with external antenna, or a ground-side ESP32 bridging UART→WiFi as a relay.
⚠ CAVEATS & GOTCHAS:
· Web Serial requires Chrome or Edge — Firefox does not support it.
· WebSocket from browser requires ws:// (not wss://) and the page must be served over plain HTTP, not HTTPS. Use python -m http.server 8080 in the dashboard folder.
· Metal airframe significantly attenuates 2.4GHz WiFi. Route an external antenna or use a fiberglass nose section.
· Wireless adds ~1ms latency. At 10Hz packet rate this is negligible.
· Close PlatformIO's Serial Monitor before opening the wired connection — only one process can hold the COM port.
· Both modes parse the identical line format — switching modes requires only the dashboard toggle, zero firmware changes.
